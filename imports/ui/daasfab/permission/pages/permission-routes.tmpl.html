Route Permissions

Blocking Routes

In order to block routes from being accessed when a usre doesn't have permission just add the following code to your $stateProvider declaration inside your config file.

For example

my-module.config.js
$stateProvider .state('triangular.my-page', { url: '/mypage', templateUrl: 'app/my-module/my-page.tmpl.html', data: { permissions: { only: ['viewMyPage'] } } });

So now if any user that doesn't have the permission viewMyPage tries to access /mypage in the browser they will be redirected to the 401 Page

Hiding menus

As well as blocking the route you will also want to hide the menu item. This is also easy, just add a permission to the menu item when you add it in your config file.

my-module.config.js
triMenuProvider.addMenu({ name: 'My Page', type: 'link', permission: 'viewMyPage', });

Now unless the user has the viewMyPage permission they will not see the menu item in the left sidebar.
